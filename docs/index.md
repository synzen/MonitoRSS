## Project Overview

**MonitoRSS** is a distributed RSS-to-Discord delivery system that:
- Fetches RSS/Atom feeds on configurable schedules
- Processes and filters articles based on user rules
- Formats messages with customizable embeds and content
- Delivers to Discord channels and webhooks

## Architecture Summary

| Component | Technology | Purpose |
|-----------|------------|---------|
| Control Panel | React, Chakra UI | User interface |
| Backend API | NestJS, Fastify | API gateway, user operations |
| feed-requests | NestJS | RSS fetching, caching |
| user-feeds | NestJS/Bun | Article processing, delivery |
| bot-presence | NestJS | Discord bot status |
| discord-rest-listener | Node.js | Discord API proxy |

**Infrastructure:** Docker Compose, RabbitMQ, MongoDB, PostgreSQL, Redis

## Core Features

1. **Feed Management**
   - Add/edit/delete RSS feeds
   - Configure message formatting (embeds, content, placeholders)
   - Set up filters for article selection
   - Schedule delivery timing
   - Clone/copy feed settings

2. **Feed Connections**
   - Discord channel connections
   - Discord webhook connections
   - Forum thread support

3. **Feed Sharing/Collaboration**
   - Management invites
   - Shared feed access

4. **Premium Features**
   - Enhanced limits
   - Priority processing

## Service Overview

### backend-api (Monolith)
- **Port:** 8000
- **Database:** MongoDB
- **Purpose:** API gateway, authentication, feed CRUD, serves Control Panel

### feed-requests
- **Port:** 5000
- **Database:** PostgreSQL, Redis
- **Purpose:** RSS fetching, response caching, rate limiting

### user-feeds / user-feeds-next
- **Port:** 5001
- **Database:** PostgreSQL
- **Purpose:** Article processing, filtering, Discord delivery

### bot-presence
- **Database:** None
- **Purpose:** Discord gateway connection, bot online status

### discord-rest-listener
- **Database:** MongoDB
- **Purpose:** Discord REST API calls with rate limiting

## Getting Started

1. Clone the repository
2. Copy `.env.example` to `.env.local`
3. Configure Discord application credentials
4. Run `docker compose -f docker-compose.dev.yml up -d`
5. Access Control Panel at `http://localhost:8000`

See [Development Guide](./development-guide.md) for detailed instructions.
